Boom Gate Model Explanation

Overview

The boom gate simulation models a single-server queueing system commonly found at car park entries, toll booths, security checkpoints, and gated communities. This page explains how individual vehicles flow through the system from arrival to departure.

System Type: Single Queue, Single Server (M/G/1 Queueing Model)
Use Cases: Car park boom gates, toll plazas, security checkpoints, parking validation stations

Key System Components

1. Vehicle Arrivals

Arrival Rate (Œª): The average number of vehicles arriving per hour (e.g., 30 cars/hr)

Arrival Pattern: Typically follows a Poisson process - random arrivals with exponentially distributed inter-arrival times

Minimum Headway: Optional minimum time between consecutive arrivals (prevents unrealistic bunching)

2. The Queue

Queue Discipline: First-In-First-Out (FIFO) - vehicles are served in arrival order

Queue Length: Unlimited capacity (vehicles can always join)

Waiting Time: Time spent in queue before service begins (zero if no wait)

3. Service Process

Single Server: One boom gate serves one vehicle at a time

Ticket Operations: Time for ticketing, payment, or validation

Movement Time: Time for boom gate to open and vehicle to move through

Total Service Time: Ticket Operations + Movement Time

4. Service Time Variability

Fixed: Same duration every time (deterministic)

Exponential: Variable duration following exponential distribution (more realistic - accounts for driver behavior variations)

Each service component can be independently configured as fixed or variable

What the Simulation Measures

Utilization Rate (œÅ)
Percentage of time the boom gate is busy serving vehicles.
œÅ = Œª / Œº, where Œº = service rate (vehicles/hour)
Average Wait Time per Arrival
Average time each vehicle spends waiting in queue before service begins (includes vehicles with zero wait).
Average Wait Time per Queued Vehicle
Average wait time for vehicles that actually had to wait (excludes vehicles that went straight to service).
Probability of Waiting
Percentage of vehicles that find the boom gate busy and must join the queue.
System State Distribution
Percentage of time the system has 0, 1, 2, ... vehicles (shows queue length patterns over time).
Hourly Maximum Distribution
Distribution of the maximum number of vehicles in the system during each hour (useful for capacity planning).

Example Scenario

  • Arrival Rate: 30 cars/hour

  • Movement Time: 10 seconds (exponential)
  • Total Average Service Time: 15 seconds

  • Utilization: ~12.5%
  • Average Wait per Arrival: ~0.2 seconds
  • Probability of Waiting: ~12%
